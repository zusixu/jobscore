## 🚀 部署前三个关键步骤

### 1️⃣ 修改配置

```bash
# 修改nginx.conf中的域名
sed -i 's/your-domain.com/example.com/g' nginx.conf

# 修改.env.production中的API地址
echo "VITE_API_BASE_URL=https://example.com" > .env.production
```

### 2️⃣ 上传到服务器

```bash
# 在本地机器执行
scp -r job-score root@your-server-ip:/opt/

# 连接到服务器
ssh root@your-server-ip
cd /opt/job-score
chmod +x deploy.sh manage.sh
```

### 3️⃣ 执行部署

```bash
# 一键部署（会自动完成所有配置）
./deploy.sh

# 或手动部署参见 DEPLOYMENT_GUIDE.md
```

## 📞 遇到问题？

### 常见问题排查

**问题1: SSL证书申请失败**
```bash
# 解决方案：确保80和443端口开放
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo certbot certonly --standalone -d your-domain.com
```

**问题2: 应用启动失败**
```bash
# 解决方案：查看日志信息
docker-compose logs
docker-compose logs job-score
```

**问题3: 访问超时**
```bash
# 解决方案：检查防火墙和网络
netstat -tulpn | grep -E ':80|:443'
curl -v https://your-domain.com
```

详见 `DEPLOYMENT_GUIDE.md` 的【故障排查】章节！
